### Задание.

1. `Описание архитектуры стенда` 
* В рамках дипломного проекта спроектирована и развернута масштабируемая, отказоустойчивая сетевая инфраструктура в облаке Yandex Cloud.
* Создание и управление ресурсами полностью автоматизировано с помощью методологии Infrastructure as Code через Terraform.
* Дальнейшая конфигурация операционных систем и развертывание программного стека автоматизированы с помощью Ansible.
 
2. `Сетевая топология и безопасность.`
* Создана изолированная сеть VPC, разделенная на публичные и приватные подсети в разных зонах доступности для обеспечения высокой отказоустойчивости.
* Безопасный доступ Bastion Host. Доступ ко всем серверам в приватном контуре осуществляется строго через выделенный шлюз bastion-host по защищенному SSH-протоколу с использованием SSH-ключей. Прямой доступ из интернета к серверам приложений закрыт.
* Группы безопасности Security Groups. На уровне облачного брандмауэра настроены жесткие правила фильтрации трафика ingress/egress, разрешающие только целевые порты для межсервисного взаимодействия.

3. `Состав и назначение хостов инфраструктуры.`
* bastion-host SSH-шлюз для безопасного администрирования приватного контура. OpenSSH Server
* web-server-1 Первый узел веб-кластера, обслуживающий запросы пользователей. Nginx, Node Exporter, Filebeat
* web-server-2 Второй узел веб-кластера (резервирование и балансировка) Nginx, Node Exporter, Filebeat
* prometheus-host Сервер централизованного сбора и хранения метрик. Prometheus Server
* grafana-host Панель визуализации состояния инфраструктуры. Grafana Enterprise
* elasticsearch-host База данных для хранения и индексации текстовых логов. Elasticsearch
* kibana-host Веб-интерфейс для анализа и поиска по логам. Kibana

4. `Описание используемого технологического стека`
* Yandex Application Load Balancer.  Внешний облачный балансировщик. Он принимает входящий HTTP-трафик от пользователей и равномерно распределяет его между web-server-1 и web-server-2. При выходе из строя одного из серверов трафик автоматически перенаправляется на исправный узел.
* Nginx: Высокопроизводительный веб-сервер, развернутый на узлах кластера для отдачи контента.

5. `Подсистема мониторинга`
* Node Exporter: Агент, установленный на каждом сервере стенда. Собирает метрики ОС CPU, RAM, дисковой подсистемы, сетевой трафик.
* Prometheus: Раз в 15 секунд опрашивает все агенты Node Exporter, собирает метрики и сохраняет их в свою Time Series Database. 
* Grafana: Подключена к Prometheus в качестве источника данных. Использует дашборд Node Exporter Full для построения интерактивных графиков, позволяющих в реальном времени оценивать здоровье инфраструктуры.

6. `Подсистема логирования`
* Filebeat: агент сбора логов на веб-серверах. Он непрерывно следит за файлами access.log и error.log веб-сервера Nginx, парсит новые строки и мгновенно отправляет их в общую базу.
* Elasticsearch: Распределенная поисковая база данных. Принимает логи от Filebeat, индексирует их и обеспечивает мгновенный поиск по терабайтам текстовых данных.
* Kibana: Панель управления над Elasticsearch. Позволяет инженерам выполнять гибкие поисковые запросы KQL и визуализировать структуру логов в реальном времени.

7. `В ходе выполнения проекта успешно решена задача развертывания инфраструктуры, соответствующей современным стандартам автоматизации и отказоустойчивости.` 
* Разработаны универсальные манифесты Terraform, позволяющие развернуть аналогичный сетевой и вычислительный контур в Yandex Cloud за несколько минут.
* Созданы отказоустойчивые плейбуки Ansible, полностью автоматизирующие установку системных пакетов, конфигурацию демонов и доставку файлов конфигурации без ручного вмешательства.
* Организована полноценная наблюдаемость  системы: метрики производительности визуализированы в Grafana, а логи доступа пользователей агрегируются в Kibana.
